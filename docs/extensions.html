**Tiny Test System** **TTS** tests behavior can be customized by different means: at runtime through a [Command line Interface](#commandlineinterface) or at compile-time by providing C++ extension points to handle custom types behavior or adapt how the whole test system is run. (insert crumbs.html here) Command line Interface ==================================================================================================== TTS supports options that can be passed as command line arguments of the test binary. Options | Shortcut | Description ------------ | -------- | ----------------------------------------------------------------------- `--count N` | `-c N` | Set size of experiments in range tests
`./my_test --count 50000` `--help` | `-h` | Display the list of supported options
`./my_test --help` `--hex` | `-x` | Display all floating points number in hexadecimal `--no-color` | `-n` | Disable the colored output of test results.
`./my_test --no-color` `--order X` | `-o X` | Change the order in which tests are run:
`./my_test --order sorted`: tests are run in alphabetical order
`./my_test --order random`: tests are run in random order `--pass` | `-p` | Display both failing and passing tests.
`./my_test --pass` `--repeat N` | `-r N` | Repeat each tests `N` times
`./my_test --repeat 42` `--seed N` | `-s N` | Provide the pseudo-random number generator seed
`./my_test --seed 13374269` `--ulpmax N` | `-u N` | Set global failure ULP threshold in range tests
`./my_test --ulpmax 4.5` `--valmax N` | `-w N` | Replace upper bound in range tests
`./my_test --valmax 10.5` `--valmin N` | `-v N` | Replace lower bound in range tests
